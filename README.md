# rara-bdd

Cucumber-rs scaffolding tool for AI agents. Generates project skeletons and step definitions from Gherkin `.feature` files — designed for AI agent workflows.

## Get Started (for Claude Code users)

**Give this prompt to your Claude Code:**

> Install rara-bdd and set up BDD testing for this project:
>
> ```bash
> cargo install --git https://github.com/rararulab/rara-bdd
> rara-bdd setup
> ```
>
> Then read the BDD section in CLAUDE.md to understand the workflow.

That's it. Your agent now knows how to write `.feature` files, generate step definitions, and verify with `cargo test --test bdd`.

## How It Works

```
.feature files  →  rara-bdd generate  →  #[given]/#[when]/#[then] skeletons
                                          ↓
                                     cargo test --test bdd (cucumber-rs)
```

1. Write Gherkin scenarios in `features/`
2. `rara-bdd generate` creates step definition skeletons with `todo!()` bodies
3. Implement the step definitions
4. `cargo test --test bdd` runs them via cucumber-rs

## Commands

```bash
rara-bdd setup                # Set up cucumber-rs in current project
rara-bdd generate             # Generate step definition skeletons
rara-bdd generate --dry-run   # Preview without writing files
rara-bdd coverage             # Check for missing step definitions
rara-bdd list                 # List features, scenarios, and steps
cargo test --test bdd         # Run BDD tests (cucumber-rs)
```

## Project Layout

```
my-project/
├── features/
│   └── auth.feature          # Gherkin scenarios
├── tests/
│   ├── bdd.rs                # cucumber-rs entry point (World struct)
│   └── steps/
│       ├── mod.rs            # Step module declarations
│       └── auth_steps.rs     # #[given]/#[when]/#[then] definitions
├── CLAUDE.md                 # BDD workflow instructions (generated by setup)
└── Cargo.toml                # cucumber + tokio in [dev-dependencies]
```
