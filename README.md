# rara-bdd

BDD testing framework for Rust projects. Maps Gherkin `.feature` files to `#[test]` functions by naming convention — designed for AI agent workflows.

## Get Started (for Claude Code users)

**Give this prompt to your Claude Code:**

> Install rara-bdd and set up BDD testing for this project:
>
> ```bash
> cargo install --git https://github.com/rararulab/rara-bdd
> rara-bdd setup
> ```
>
> Then read the BDD section in CLAUDE.md to understand the workflow.

That's it. Your agent now knows how to write `.feature` files, matching tests, and verify with `rara-bdd run`.

## How It Works

```
@AC-01 in .feature  →  fn ac_01_*() in #[test]
```

1. Write Gherkin scenarios with `@AC-XX` tags in `features/`
2. Write `#[test] fn ac_XX_*()` functions in your test files
3. `rara-bdd run` discovers, matches, and executes them

## Commands

```bash
rara-bdd setup                # Set up BDD in current project
rara-bdd run                  # Run all BDD scenarios
rara-bdd run --report json    # JSON output (for CI)
rara-bdd coverage             # Check for uncovered ACs
rara-bdd list                 # List AC ↔ test mapping
rara-bdd trace                # Generate TRACEABILITY.md
```

## Project Layout

```
my-project/
├── features/
│   ├── auth.feature          # Gherkin scenarios with @AC-XX tags
│   └── TRACEABILITY.md       # Auto-generated
├── tests/
│   └── auth.rs               # #[test] fn ac_01_*(), ac_02_*()
├── CLAUDE.md                 # BDD workflow instructions (generated by setup)
└── Cargo.toml
```
