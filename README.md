# rara-bdd

BDD testing framework for rararulab Rust projects. Parses Gherkin `.feature` files and matches acceptance criteria to `#[test]` functions by naming convention.

## Quick Start

```bash
# Install
cargo install --git https://github.com/rararulab/rara-bdd

# Run BDD suite (JSON for CI)
rara-bdd run --features-dir features --report json

# List scenarios with matched tests
rara-bdd list --features-dir features

# Check coverage gaps
rara-bdd coverage --features-dir features

# Generate traceability matrix
rara-bdd trace --features-dir features
```

## How It Works

```
@AC-01 in .feature  -->  fn ac_01_*() in #[test]
```

1. Write Gherkin scenarios with `@AC-XX` tags in `features/`
2. Write `#[test] fn ac_XX_*()` functions in your test files
3. rara-bdd discovers and matches them automatically

## Per-Project Setup

```
my-project/
├── features/
│   ├── auth.feature          # Gherkin scenarios with @AC-XX tags
│   └── TRACEABILITY.md       # Auto-generated by `rara-bdd trace`
├── src/
├── tests/
│   └── auth.rs               # #[test] fn ac_01_*(), ac_02_*()
└── Cargo.toml
```
